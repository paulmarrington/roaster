Reason - notes of why this project is needed

uSDLC2 required a light-weight self-installing application server. Originally built-in, I refactored it into a separate package called roaster because it suited the needs of other applications I was working on.

Investigation - the path to a decision

I wanted a system that would handle web-sockets and streaming as well as HTML. It should also be a small basic install with only modules required to be installed on demand. As uSDLC is a one-page application, complex templating and application server frameworks are not required. In this repect even Django can be considered heavy-weight. I am sure there are others available, but the two main contenders I knew of were twisted and node-js. Twisted is an excellent lightweight TCP/HTTP solution and Python a good language (although the lack of easy closures is a limitation I would prefer not to have). In the end Node-JS is supported on more platforms, is easier to install and has a more vibrant support and extension community.

Result - a high level description of what the finished project will look like

A light-weight application server with the following features:

  • Installation from a single script that fetches required components from the Internet
  • A package-on-demand system that will only download lesser-used packages when they are first needed.
  • In-built pub-sub support
  • Serving static resources
  • Serving dynamic resources in javascript, coffee-script, live-script or Kaffeine or from external applications

User Interface - technology, look and feel, user experience

Roaster does not have a user interface - it is concerned with providing efficient server for differing user interfaces. It is designed to effectively support single-page browser applications.

Language - computer languages and libraries decided upon

The client browser and the server (node-js) use javascript as the native language. Roaster also support languages that compile to Javascript - such as coffee-script, live-script and Kaffeine. Roaster's JIT compiler also supports other related languages such as Less and Stylus for CSS. Server side execution can use other languages that are run as scripts or application servers.

Deployment - envisioned deployment environment

Roaster is installed with a single system-dependent script - one for Windows and one for *nix. It will download and unpack roaster into the target directory. It will then download and install Node and the core requisite node packages. Any systems built with roaster will wrap these scripts on others for the larger application.

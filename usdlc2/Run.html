Running Roaster Applications

Debugging

Stand-alone roaster applications

For anything but a server, set an environment variable $DEBUG_NODE to debug,  --debug or --debug-brk. The former will trigger the node internal debugger interface, while the latter work with node-inspector. Install node-inspector if you have not run a roaster server in debug mode.

Ref: go.shdirs.jsserver.coffeesteps.coffee

Server-based roaster applications

From any roaster based server, add config=debug to the command line:

./go.sh server config=debug

This will download node-inspector if necessary and start the server in debug mode. As per instructions the debugger can be accessed via chrome (being a current node-inspector requirement) with

http://127.0.0.1:8080/debug?port=5858
Ref: server.coffeego.shprocesses.coffeeterminate.coffee

Updating Running Server

Roaster applications in debug mode do not attempt to restart on every source change. Instead they provide simple mechanisms for manual restart. In debug mode there is a restart link on the top right. Additionally Esc-Esc will restart server and client.
